Тип ответа SearchResponse

interface SearchResponse {
  answer:  string;                   // LLM-ответ на языке запроса
  results: SearchResultWithStore[];  // список товаров
  basket?: BasketResult;            // только для basket-режима
}

interface SearchResultWithStore {
  id: string; storeId: string; storeName: string;
  normalizedName: string; nameHe: string|null; nameRu: string|null; nameEn: string|null;
  category: string|null; price: number|null; quantity: number|null;
  unit: string|null; barcode: string|null;
  similarity: number;           // 0..1 косинусное сходство
  distanceKm?: number;          // если переданы координаты
  storeLat?: number|null;       // для Waze / Google Maps
  storeLng?: number|null;
  previousPrice?: number|null;  // зарезервировано под price_history
}

interface BasketResult {
  items: string[];                     // пункты корзины
  storeOptions: BasketStoreOption[];  // до 5, отсортировано
  savings: number;                    // экономия (₪) между лучшим и худшим вариантом
  bestStoreId: string|null;
}

POST /api/ingest — приём данных из POS

// Body:
{
  storeId:  "uuid",   // UUID магазина — общий ключ B2B↔B2C
  provider: "morning" | "green_invoice" | "manual" | "other",
  products: [{ id, name, price, quantity?, unit?, barcode? }]  // макс. 500
}

// Ответ:
{ success: true, store: { id, name }, processed: N, upserted: N, skipped: N }

Пайплайн ingest

1. Validate (Zod schema)
2. Verify store exists (Supabase)
3. Compute sha256 per product
4. Fetch existing hashes → filter changed
5. normalizeProducts (GPT-4o-mini, batch 30)
6. generateEmbeddingsBatch (batch)
7. Upsert (ON CONFLICT store_id, pos_item_id)

Умная инкрементальная синхронизация

Если payload товара не изменился (sha256 совпадает) — LLM и embedding не вызываются. Только изменённые товары проходят через AI-пайплайн. Это критично для экономии OpenAI-кредитов при повторной загрузке прайса.

Семантический фильтр намерений (intentCheck)

LangChain-цепочка перед каждым поиском проверяет, что запрос — это реальный поиск товара, а не разговорная фраза или мусор:

PromptTemplate  →  ChatOpenAI('gpt-4o-mini', temperature=0)  →  StringOutputParser
// возвращает 'YES' или 'NO'

Redis кэш

Ключ: intent:<sha256(query)>
TTL: 24 часа
Значение: 'YES' / 'NO'

Одинаковый запрос → OpenAI вызывается только 1 раз в сутки

Langfuse трассировка

userId = IP клиента
metadata = { query, route }
Flush — fire-and-forget

Не блокирует ответ API

AI / LLM слой (openai.ts)

Модели

gpt-4o-mini — нормализация + генерация ответов

text-embedding-3-small (1536 dim) — семантические эмбеддинги

Батчинг

Нормализация: batch 30 товаров / вызов

Эмбеддинги: весь массив за 1 вызов API (до 2048 входов)

Нормализация (normalizeProducts)

GPT-4o-mini генерирует JSON с полями: nameHe, nameRu, nameEn, category, unit. Эмбеддинг строится из конкатенации всех трёх языков — запросы на иврите, русском и английском попадают в один вектор.

Персона AI-ответа ("Аchi Sababa")

Все ответы поиска генерируются через единый PERSONA_PROMPT — "אחי הסבבה" (Savvy Israeli Neighbor): тёплый тон, израильский слэнг, язык ответа = язык запроса. Включает правило дефицита: если quantity > 0 и < 5 → обязательное предупреждение.

Ключевые точки интеграции B2B → B2C

Общий ключ: storeId (UUID)

B2B знает storeId магазина. B2C использует тот же UUID как store_id во всех таблицах. Запрос GET /api/search?store=<uuid> фильтрует результаты по конкретному магазину.

Точка обновления цен в реальном времени

B2B нажимает "Обновить цену" → POST /api/ingest → upsert в products. При добавлении Supabase Realtime B2C-клиент получит обновление без перезагрузки. Redis-кэш нужно инвалидировать при изменении цены.

Что нужно добавить для полной реализации Фазы 3

  1. Supabase Realtime подписка на products table в клиенте → обновление BasketState при изменении цены
  2. Инвалидация Redis search:cache:* при POST /api/ingest (по store_id)
  3. Таблица price_history в Supabase + заполнение previousPrice в SearchResultWithStore
  4. QR-код на PDF-ценниках (B2B) ведёт на /api/search?store=<uuid>&q=<product>
